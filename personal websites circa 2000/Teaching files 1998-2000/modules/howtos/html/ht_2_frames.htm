HTML Framesets

Framesets: Introduction

Framesets are html documents that tell a browser how to display other document in their own seperate panels.

Framesets have some advantages:

    • They assist in navigtation as navigation iinformation does not have to scroll with a content page
    • They can make a site look good
    • Only one page loads at a time, which is visually less tiring
    • There are less pages to update if navigation changes

There are some disadvantages:

  • It's can be very hard to link from one content page to another
  • Some (very) old browsers and newer devices like handhelds (palm pilots and similar) don't like them much
  • They have to be planned carefully to make sure they don't take up too much space
  • Because the structure is more confusing, updating pages can be a headache
  • Most database driven sites avoid frames

Framesets: The basic idea

<html>
<head>
<title>My First Frameset</title>
</head>

<frameset rows="100,*">
<frame src="top.htm">
<frame src="main.htm">
</frameset>

<noframes><body bgcolor="#FFFFFF">
This site uses frames...
</body></noframes>
</html>

This html document will load two other documents (top.htm and main.htm) into two rows in the browser window. The top pane will 100 pixels deep and the lower one will take the rest of the browser window.

Do this:

  • Copy the above code into a new page and save it as frame.htm.
  • In the same directory, make two pages, one called top.htm and another called main.htm and save them.
  • Open frame.htm into a browser

There's a few things to note about this frameset:

  • There are (ugly) 3d borders displayed
  • The user can freely drag the border up and down

Using rows and cols

You can specify rows or cols for each <frameset> tag, not both.

Ther allowable units of measure are:

pixels: rows=200, *"
percentages: rows="20%,*"
the * just tells the browser to use up the available space

You get a fixed-width border at the bottom of the page by: rows="*, 120"

You can use as many rows or cols as you want: cols="150, *, 100" give you two fixed width panels, one at the top and one at the bottom, and a free-sized centre panel. You must match the number of rows or cols with <frame></frame> tag sets.

You can also add a border color (rendered by IE only): bordercolor="#ffggbb"

Further Attributes of the <frame> tag:

frameborder=1|0 turns borders on or off
marginheight=n and marginwidth=n act as they do in a body tag
name="whatever" give the panel a target name you use to link to content
noresize disables the users ability to drag the borders
scrolling=yes|no|auto controls how clipped content behaves
src="url" tells the browser which page starts inside that panel of the frameset

Do this: add the following information to your frameset to make it behave better:

<html>
<head>
<title>My Second Frameset</title>
</head>

<frameset rows="100,*">
<frame src="top.htm" noresize marginheight=0 marginwidth=0 scrolling="auto">
<frame src="main.htm" noresize marginheight=0 marginwidth=0 scrolling="auto">
</frameset>

<noframes><body bgcolor="#FFFFFF">
This site uses frames...
</body></noframes>
</html>

Nesting Framesets

More complex framed layouts are achieved by nesting framesets, just like you do with tables: You will need to make another page called 'left.htm' for this to work.

<html>
<head>
<title>My Third Frameset</title>
</head>

<frameset rows="100,*" frameborder=0>
<frame src="top.htm" noresize marginheight=0 marginwidth=0 scrolling="no">
<frameset cols="150,*" frameborder=0>
<frame src="left.htm" noresize marginheight=0 marginwidth=0 scrolling="no">
<frame src="main.htm" noresize marginheight=0 marginwidth=0 scrolling="auto">
</frameset>
</frameset>

<noframes><body bgcolor="#FFFFFF">
This site uses frames...
</body></noframes>
</html>

Some things to note about this frameset:

We added the frameborder=0 into the <frameset> tag to show borders being rendered.

We set two windows to not scroll, and the main page to scroll only if there was clipping (auto)

Using the name attribute to link pages from one frame to another:

We set the name attribute so that we can use the target attribute inside the <a href> tag:

<html>
<head>
<title>My Third Frameset</title>
</head>

<frameset rows="100,*" frameborder=0>
<frame src="top.htm" noresize marginheight=0 marginwidth=0 scrolling="no">
<frameset cols="150,*" frameborder=0 name="top">
<frame src="left.htm" noresize marginheight=0 marginwidth=0 scrolling="no" name="left">
<frame src="main.htm" noresize marginheight=0 marginwidth=0 scrolling="auto" name="main">
</frameset>
</frameset>

<noframes><body bgcolor="#FFFFFF">
This site uses frames...
</body></noframes>
</html>

Now we can control what page loads into the main window from the side window:

In side.htm doing this:

<a href="newpage.htm" target="main">Load a new page</a>

will result in a link that loads "newpage.htm"into the main window and not the side window. This goes all ways, so we can target any frame from any other.
