Fireworks - Basic Image Optimisation

Last time we learnt how to add a frame to a fireworks document. then we exported the frames individually from Fireworks, then added a rollover in Dreamweaver. This is fine, but consider the following two tasks:

  • A client wants a really large image on the front page of a site, say around 80k or
  • You want to make a whole set of rollovers images for a navigation page in a frameset, complete with extra rollovers

This where Fireworks really comes into it's own. As well as generating frames, FW can also 'slice' images and write it's own html pages... excellent!

Slicing

Slicing is a term for the practise of cutting an image up into pieces and then reassembling it in a table. There are several advantages to slicing images:

  • Servers can stream multiple small images faster than a single larger one
  • You can use different commpression methods and settings for different parts of a larger image
  • You can delete parts of flat colour in an image

1. Make or open a largish image in Fireworks.

Make sure you have the 'Show Slices' toggle on:

It's in the lower orange circle:

2. Select the Slice tool.

It's the other orange circle.

3. Use the Slice Tool

It's a rectangular tool that works just like the box drawing tool. Click and drag to use. When you drop, you will see the a green rectabgle overlaid on the image, this is the 'slice'and tells FW where you would like the image to be cut. The red lines that appear show you the shape of the table that FW is going to generate.

Continue slicing until you have covered the image. Bear in mind the follwoing points:

  • A large area of flat color can be sliced and later deleted to cut down on download time - but only if the bgcolor of the html page is the same as the deleted slice
  • Slice to make areas that will export better with different types of ceompression; ie photographic areas will be better as jpegs and crisp text better as gif

You can use the sub-select tool (white arrow) to reshape your slices after you have drawn them.

4. Optimise each slice

Use the select tool to pick up each slice and optimise each slice as you go.

5. Save your file

The slice information is stored on the 'web layer' and is saved with your file

6. Export the file

In the resulting dialog box you get a number of options:

Save In is the directory where you want the image files to be saved to. It should be in a folder inside your DW root directory

Because FW will generate a number of sperate image files, it needs a Base Name. It will use this to name the files it makes (ie the images in this image will be exported as academica_r1_c1.gif for row 1 column 1 and so on)

In Slicing select 'Use Slice Objects' to make FW cut the image up

In the HTML box:

Style selects the way that FW will write the html page. Choose Dreamweaver 3.

Location selects where the HTML page will be written (NOT the images!) It's important to get this right as FW will write a page with all the folder relationships for you (../ and so on).

If you get a dialog that says: This object contains overlapping slice objects, STOP! Go back to the image and make sure all the slices are correctly aligned. Switch snap to guides on (in the view menu) to increase your chances of getting it right.

Click ok and open the html page FW has written in DW. Ole!

If you have areas of flat colour you can delete them in DW or in FW select "Text" in image type on the optimise palette
