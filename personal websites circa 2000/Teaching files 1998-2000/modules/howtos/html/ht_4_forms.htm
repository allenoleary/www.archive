HTML Forms

Forms are a way to collect informaiton from a user. This could be just about anything you can type, for any reason. It's common to use forms to enable site searches, collect passwords, pay bills and so on. Forms have even been used...

There are several types of form elements:

the text field, single line

the password field (go on type something into it...)

the multi-line text box

the select-menu field (one line displayed, one selection only)

the checkbox field (multiple yes/no's allowed)

the radio button field (one only selected at a time)

the Submit and Reset buttons (click the reset to clear all entries)

All of the above fields are kept in one <form> </form> tag pair. Everything in a <form></form> tag is alogical group, that is if this were information to be sent to the server, it would all be sent as one group. It doesn't matter how much much or how little information or formatting is inside a form tag, the useage is just about how much information you want to gather at once. The <form> tag accepts no formatting attributes.

The <form> tag.

The form tag only accepts two useful attribute - the first is <form action="http://www.search.com/find.cgi">. The value here is a program on the server that will do something useful (like add you to a mailing list or database).

The second is <form action="http://www.search.com/find.cgi" method="post">. Post and Get are the only two values accepted. Post is more common, it sends a data stream to the server, where Get sends a url with an attached string, (like http://www.search.com/find.cgi?alotronic) which is processed by the server. This is common more common with search engines than other applications (have a look at a search string from looksmart for instance) because they can be bookmarked by the browser.

The input elements

Input elements define the fields, what gets displayed and also what name/value pair is sent to the server. Name/Value pairs are the name and value of the variable that gets sent to the server. Again, have a look at looksmart to see how name/value paris are encoded when sent (with an & between name/value pairs).

Textbox

The name and value attributes are shared by all form elements, but getting information abot what is actually selected is different for some of the input elements. The basic textbox input field looks like this:

<input type=text name=country size=20>

In a working form it would work like this:

<form action="http://www.search.com/find.cgi" method="get">

<p>search for <input type=text name=search size=20>
<p>in what country ?<input type=text name=country size=20>

<p><input type=submit>

</form>

this gives you:

search for

in what country ?

This would send an encoded URL to the server which would look something like this: http://www.search.com/find.cgi??search=alotronic&country=australia.

Note that the size attribute is how many characters will display, but the user can enter any length of string that they want.

TextArea

Gives you larger boxes over multiple lines for more inout that would be comfortable with a textbox.

<TEXTAREA ROWS="4" COLS="40">
</TEXTAREA>

This will give you a textarea 4 rows by 20 columns of charcters (depending on the browser default):

You can add default text to a textbox, which can be edited by the user:

<TEXTAREA ROWS="4" COLS="40">
Enter a complete description of your problem.

Please include a phone number where you can be
reached for more information if necessary.
</TEXTAREA>

Password

<input type=password name=secret size=15>

Works exactly the same way as textbox, but adds the * to keep your entry private.

CheckBox

<input type=checkbox name="check" value="yes" checked>

This information is only sent to the server if the element is checked. You use the checked attribute to set the checkbox to be selected when the page loads. You must supply a name and a value for this element.

Any number of checkboxes can be selected at once, but if you want to send several bits of information at once, you make the name attribute the same, but alter the values:

<P>Which pets do you own?
<P>

<INPUT TYPE="checkbox" NAME="pets" VALUE="dog" checked> Dog
<BR>
<INPUT TYPE="checkbox" NAME="pets" VALUE="cat"> Cat
<BR>
<INPUT TYPE="checkbox" NAME="pets" VALUE="fish"> Fish
<BR>
<INPUT TYPE="checkbox" NAME="pets" VALUE="other"> Other

</P>

Gives this:

Which pets do you own?

Dog
Cat
Fish
Other

Radio Buttons

Where Checkboxes allow (and send) multiple options, Radio Buttons will send only one. The basic element looks like this:

<INPUT TYPE="radio" NAME="shipping">

But to get them to work as a group, you have to give them the same name. The value specifies whar will be sent to the server - just the one that is selected

<P>How should we ship your order?
<P>
<INPUT TYPE="radio" NAME="shipping" value="next day" > Next day air
<BR>
<INPUT TYPE="radio" NAME="shipping" value="second day" > Second day air
<BR>
<INPUT TYPE="radio" NAME="shipping" value="courier"> Courier
<BR>
<INPUT TYPE="radio" NAME="shipping" value="post"> Parcel post
</P>

How should we ship your order?

Next day air
Second day air
Courier
Parcel post

The Select field:

This is the one that gives you the basic drop-down list. Again, only the selected option value is sen tot eh server.

<SELECT NAME="shipping">
<OPTION VALUE="Next AM">Next day, AM delivery</OPTION>
<OPTION VALUE="Next PM">Next day, PM delivery</OPTION>
<OPTION VALUE="Second">Second day</OPTION>
<OPTION VALUE="Parcel">Parcel post</OPTION>
</SELECT>

Note that adding the selected attribute make that option come up in the textbox...

<SELECT NAME="shipping">
<OPTION VALUE="Next AM">Next day, AM delivery</OPTION>
<OPTION VALUE="Next PM">Next day, PM delivery</OPTION>
<OPTION VALUE="Second" SELECTED>Second day</OPTION>
<OPTION VALUE="Parcel">Parcel post</OPTION>
</SELECT>

And finally...

The Submit and Reset Buttons:

<input type="submit" name="Submit" value="Submit">
<input type="reset" name="Reset" value="Reset">

You can change the text on the buttons by changing the Value.

<input type="submit" name="Submit" value="Submit to My Will!!!">
<input type="reset" name="Reset" value="Reset my brain ">
