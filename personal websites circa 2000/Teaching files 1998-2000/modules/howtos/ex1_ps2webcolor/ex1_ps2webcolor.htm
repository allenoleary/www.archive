Getting Images for the web out of Photoshop

Structure of the file types and uses

    gifs: Usually used for 'flat color' and crisp-edged graphics. Also where animation and transparency are required.

    • max 256 colors
    • support transparency
    • animation
    • lossless compression
    • uses indexed color palettes to select colors from original image
    • interlacing ability

     

    jpegs: Usually used for soft-edged images, particularly photos where the compression algorithm works wonders on file sizes.

    • max 16.4 million colors
    • lossy compression (throws picture information away)
    • compresses on a percentage basic
    • progressive ability

     

Photoshop and Web Imaging

Thankfully Photoshop 5 has web palettes built in! If you have Photoshop 5.5 or above you get 'Image Ready' with it which functions much like Fireworks and has much finer control over sizing and options. However for this exercise we'll assume you've just got PS 5

Getting gifs:

1. You must convert your file to indexed color before you export it (Image/Mode menu)

You'll get a dialog box like this:

In the Palette Menu the three choices we're concerned with are:

  1. exact - if there's less than 256 colors in the source image use this to force exactly the same colors into the gif
  2. web - forces the palette in the 216 web-safe color palette
  3. adaptive - chooses the best 256 colors in the image with no concessions to the web

As you select each one, you'll see your image change considerably. If you've made the effort to design the graphic with web-safe colors, use web or exact. Adaptive gives the best results, but may look bad on older systems.

The Dither options relate to how PS mixes the colors together if the image is very complex and uses a lot more that 256 colors. Try selecting some of these options to preview the results. This can be very messy. If you're using an image with this many colors and you don't need transparency or animation, convert it to a jpeg instead.

2. Use "Save as..." to save the current image as a gif

    • Select "CompuServe GIF" from the drop down menu. (CompuServe own the algorithym)
    • Give it a file name and make sure you add the right extension (filename.gif)
    • Click OK.
    • Yet another dialog box will appear asking you whether you want 'Normal'or 'Interlaced' row order. If you select 'Interlaced' the image will appear in chunks which define over time. This adds a little file size and isn't widely used today as modems are much faster than they were. Click OK.

Getting jpegs:

1. You must convert your file to RGB before saving as a jpeg (Image/Mode menu)

2. Use "Save As..." to give the file a name and location. Again it's VERY important to add the .jpg file extension to the image name. Don't, whatever you do, save over your original as you will lose that information for ever...

3. Now you'll be presented with the compression dialog box:

The best setting for quality is usually 8. This gives a good balance between file size and quality. The file exported at 8 will typically be half the size of one exported at 10, and will look almost identical.

The 'progressive' option modes the image 'fade in', again, not used much these days, so leave it at baseline.

3. Click OK
