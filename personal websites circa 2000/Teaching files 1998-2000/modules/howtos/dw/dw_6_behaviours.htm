Dreamweaver Behviours

Introduction:

Dreamweaver behaviours are just javascripts of various levels of cleverness that Dreamweaver developers have written. There is no functional difference between a behaviour inserted by dreameaver and the same thing copy and pasted as a javascript in a text editor - it all uses the same basic building blocks.

The good thing aout behaviours is that they are pretty much bomb-proof and are fairly reliable across most browsers and platforms, and Dreamweaver wraps it all up nicely for you so that you don't have to get your hands dirty with the javascript.

Some behviours (such as 'Open Browser window' are fairly basic and easy to understand. Others are very complex and beyond the understabding of most web publishers. However it is good to have a basic understanding of how javascript functions to use them well.

JavaScript: Just the basics.

JS is a way of the page telling the browser what to do with the loaded page. Some javascript functions emulate what a user might do with a browser like this: document.history.back(1). This operates just as if you have hit the back button on our browser. Other javaScript might collect information about the environment the page is in, such as what platform or browser is currently viewing the document (called a sniffer script). Javascript can also handle more difficult things like layer animation or accomplishing more than one thing at once (like opening two new ages into different panels of a frameset at once).

What makes the javascript work?

Javascript can work in several ways:

  • You can use some javascripts in the body of a page to write conditional html

  • You can call javascript directly from a <a href> tag, where it is attached to an event

  • You can call reusable javascript functions from an <a href> tag where it is attached to an event.

See the difference between the last two

In Dreamweaver, there is pretty much always a function. This has the advantage that you can use a single script more than once in a page, so it cuts down page sizes and makes you files easier to read.

What's an event then?

An event is a way that a browser catches user actions. For example, even though you don;t see it, there is an implied event when you click a plain old text link: onClick. What event can be caught where depends on the browser, but some of the common ones are:

<a href="#" onMouseOver="doSomeJavaScript( )" onMouseOut="doSomethingelse( )">

<body onLoad="doSomeMoreJavaScript( )">

For the body tags the most useful events are:

onLoad: when a page starts loading
onBlur: When the user clicks off the current page
onFocus: When a user clicks onto the current page

Otherwise the most common are:

OnClick
OnMouseOver
OnMouseOut

There are also more esoteric events (especially in IE 4+) like: onHelp
