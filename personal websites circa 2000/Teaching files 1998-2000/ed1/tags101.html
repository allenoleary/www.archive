HTML 3.2 The Basic Tags

Compatibility

To make good looking pages you don't really need a lot more than these. These are the tags that have really done the work over the last couple of years. HTML 4 is a much more evolved and complex beast, but a lot of it is only supported by 4+ browsers. The tags in this document are those which are supported by both IE3 and Netscape 3. These browsers both exceed and fall short of the HTML 3.2 standard in various areas.

Physical Text Formatting tags.

These tags are called `physical' because they affect the way the content is displayed, but do not provide extra information about the content. For instance the physical <B>bold</B> tag bolds the text, and so does the logical <STRONG>strong emphasis</STRONG> tag, but on a browser that converted text to speech for the partially sighted the latter would be more useful as the content is then read slightly more loudly to the computer user.

Tables.

Tables started out being good ways to arrange tabular information, but the use of them is now just as common as a way to format the layout of the page, or to 'slice' images. They can be tricky to keep hold of in code when they get large, but really provide a de-facto grid layout system, specially when the BORDER attribute is set to '0' (off). Tables can be nested. It's worth experimenting closely with the COLSPAN and ROWSPAN tags.

Tag

Closing Tag

</TABLE>

Linking

These tags define the ways to link to other documents, or move within a single document.

Tag

Closing Tag

Attributes

Notes

<A >

</A>

HREF="url" (absolute)
HREF="path" (relative)

The HREF (required) can be expressed as a relative, absolute or local link. The NAME attribute specifies a link inside a document that is linked to with the # symbol in the HREF attribute. The <A>tags can be used around text or images - se the BORDER=0 to remove the highlighted border from around the linked image.

Framesets

Framesets really are the bridge from basic to intermediate HTML, as we need to begin to get to grips with the concepts of how the browser deals with documents as groups of objects. A frameset is specified in it's own variation on the HTML document, where the <FRAMESET> tag is used in place of the <BODY> tag. Within the <FRAMESET> tags, the <FRAME> tag points to another document that is contained within the frameset. It can be confusing, but is vital to master if you wan to provide pages with navigation sidebars. Also worth noting is that some browsers will have trouble printing from frames.

Tag
