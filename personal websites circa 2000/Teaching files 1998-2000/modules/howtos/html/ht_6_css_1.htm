Style Sheets 1: The Basics

The Idea

Style sheets are a return to the one of the basic tenets of the internet - the seperation of presentation and content. Whiile initially this was a move to stop the proliferation of propprietry tags from Netscape and Microsoft, style-sheets are now extremely important that the web is atarting to be accessed from an increasing variety of hand-held devices, TV's, game consoles and so on.

The Positives

Style sheets mean that all pages on a site can read just one set of information about presentation. This means that changing a single text-file can affect the look of hundreds of pages. JavaScript can also be used to link to different satyle-sheets depending on the viewing device, screen width and so on. It also eliminates the <font> tag and a significant amount of size from documents, and makes them a lot easier to read.

Increased typographic control is also possible, including text-borders, line-height, margin setting, setting absolute positions of items and so on.

The Negatives

Despite the existence of Style sheets since 1996 browser support is still very patchy. The only completely compliant browser in Opera 4. Mavigatoe 6 is also rumoured to be very good. But of the current crop, not even IE 5 observes all the basics.

Text-size is notoriously hard to control, and even though you can theoretically specify point sizes, pixel-heights, and html sizes only pixel sizes are reliable across all platforms and browsers. Some of the finer points of control just don't work a lot of the time.

How it works

There are two basic ways style sheets work: Redefined html and custom classes. Redefining html means that you deploy the basic html tags in an 'old-fashioned' way and then tell the browser how to display this information:

<html>
<head>
<title>Style Sheet Example</title>
<style type="text/css">
<!--
body { background-color: #FFFFCC; margin-top: 18px; margin-right: 100px; margin-left: 100px}
p { font-family: Verdana, Arial, Helvetica, sans-serif; font-size: 12px; color: #000033; line-height: 150%}
h1 { font-family: Verdana, Arial, Helvetica, sans-serif; font-size: 20px; font-weight: bold; color: #000033}
-->
</style>
</head>

<body bgcolor="#FFFFFF">

<h1>This is a normal heading</h1>
<p>This is just paragraph text and shows how the margin for the whole page is controlled as well as how the line-height has been altered as well to 150%</p>
<p>This is another paragraph.</p>

</body>
</html>

Specifying Font and Color Properties

You specify font-properties for all the html font-related tag you are altering from the default:

'font-family'

Unlike most other CSS1 properties, values are separated by a comma to indicate that they are alternatives:

BODY { font-family: gill, 'times new roman', sans-serif }

'font-style'

Value: normal | italic | oblique

H1, H2, H3 { font-style: italic }

Using oblique is not very reliable

'font-variant'

Value: normal | small-caps

H3 { font-variant: small-caps }

'font-weight'

Value: normal | bold | bolder | lighter | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900

The 'font-weight' property selects the weight of the font. The values '100' to '900' form an ordered sequence, where each number indicates a weight that is at least as dark as its predecessor. The keyword 'normal' is synonymous with '400', and 'bold' is synonymous with '700'. Keywords other than 'normal' and 'bold' have been shown to be often confused with font names and a numerical scale was therefore chosen for the 9-value list.

P { font-weight: normal }

H1 { font-weight: 700 }

'font-size'

absolute (px, pts) relative (ems, %, small | medium | large )

Using absolute sizes is officially discouraged, and there is a big problem on the mac where points are conveted to pixels (so a value of 9 would render text illegible). The only reliable absolute measure is pixels. Of course the user can change the size of the text on the page by using the 'text-size' command, so you will never have total control.

Relative sizes suffer from the same faults as normal html text (mac renders only 70% of PC size). Medium is the same as a html size of 3. However the relative measure can be useful as text-change of other elements will scale with any user made text sixe changes.

'color'

Value:

This property describes the text color of an element (often referred to as the foreground color). There are different ways to specify red:

EM { color: red }
EM { color: rgb(255,0,0) }
Em { color: #00CCC }

'background-color'

Value: | transparent

Applies to: all elements

This property sets the background color of an element.

H1 { background-color: #F00 }

'background-image'

Value: URL | none

This property sets the background image of an element. When setting a background image, one should also set a background color that will be used when the image is unavailable. When the image is available, it is overlaid on top of the background color.

BODY { background-image: url(background.jpg) }

background-repeat'

Value: repeat | repeat-x | repeat-y | no-repeat

If a background image is specified, the value of 'background-repeat' determines how/if the image is repeated. A value of 'repeat' means that the image is repeated both horizontally and vertically. The 'repeat-x' ('repeat-y') value makes the image repeat horizontally (vertically), to create a single band of images from one side to the other. With a value of 'no-repeat', the image is not repeated.

BODY { background: red; background-image: url(background.jpg) ; background-repeat: repeat-y; }

In the example above, the image will only be repeated vertically.

'background-attachment'

Value: scroll | fixed

Specifies whether a background image scrolls with the content or remains fixed in position.

The above style info can be pulled together like this:

<style type="text/css">
<!--
body { background-color: #FFFFCC; margin-top: 18px; margin-right: 100px; margin-left: 100px}
p { font-family: Verdana, Arial, Helvetica, sans-serif; font-size: small; color: #000033; line-height: 150%}
h1 { font-family: Verdana, Arial, Helvetica, sans-serif; font-size:120% font-weight: bold; color: #000066}
-->
</style>

Controlling further text properties:

'text-align'

Value: left | right | center | justify

'text-indent'

Use pxs, % or ems.

The property specifies the indentation that appears before the first formatted line. An indentation is not inserted in the middle of an element that was broken by another (such as 'BR' in HTML). Example:

P { text-indent: 3em }

'line-height'

Value use % or ems

The property sets the distance between two adjacent lines' baselines.

'text-transform'

Value: capitalize | uppercase | lowercase | none

'capitalize' uppercases the first character of each word 'uppercase' uppercases all letters of the element 'lowercase' lowercases all letters of the element 'none' neutralizes inherited value.

'text-decoration'

Value: none | [ underline || overline || line-through || blink ]

This property describes decorations that are added to the text of an element. If the element has no text (e.g. the 'IMG' element in HTML) or is an empty element (e.g. ''), this property has no effect. A value of 'blink' causes the text to blink.
